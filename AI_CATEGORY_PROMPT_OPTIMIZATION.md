# AI 智能分组提示词优化说明

## 🎯 优化目标

提升 AI 智能分组的识别准确率，特别是对于知名网站（如 3dm、github、bilibili 等）的分类准确性。

## 📊 问题分析

### 优化前的问题

**原始提示词：**
```
你是一个智能分类专家。请为以下书签推荐最合适的分组。
标题：{title}
网址：{url}
域名：{domain}
```

**存在的问题：**
1. ❌ **缺少分类知识库** - AI 不知道 3dm 是游戏网站
2. ❌ **没有分类体系** - AI 自由发挥，结果不稳定
3. ❌ **没有判断优先级** - AI 不知道如何利用域名信息
4. ❌ **分类标准不统一** - 不同 AI 可能给出不同结果

### 实际案例

```
输入：
- 标题：3DM游戏网
- 网址：https://www.3dmgame.com
- 域名：3dmgame.com

优化前结果：其他 ❌
优化后结果：游戏资讯 ✅
```

## 🔧 优化方案

### 核心改进

#### 1. **添加分类知识库**

建立了 12 大类、50+ 细分类型的网站分类知识库：

```
🎮 游戏类：
- 游戏媒体：3dm、游民星空、游侠网 → 游戏资讯
- 游戏下载：steam、epic、gog → 游戏平台
- 游戏社区：nga、tieba → 游戏社区

💻 开发技术类：
- 代码托管：github、gitlab → 代码托管
- 技术文档：mdn、devdocs → 技术文档
- 开发工具：vscode、jetbrains → 开发工具
- AI/ML：huggingface、kaggle → 人工智能

📚 学习资源类：
- 在线教育：coursera、udemy → 在线教育
- 知识百科：wikipedia、百度百科 → 知识百科
- 学术论文：arxiv、cnki → 学术论文

... 更多分类
```

#### 2. **明确分类规则**

```
【分类规则】
1. 优先使用域名判断（3dmgame.com → 游戏资讯）
2. 其次参考标题关键词（GitHub → 代码托管）
3. 参考已有分组，尽量归类到现有分组
4. 分组名称：2-4个中文字，简洁明了
5. 如果无法匹配任何类型，输出"其他"
```

#### 3. **结构化提示词**

采用清晰的层次结构：
- 【任务】- 明确要做什么
- 【输入信息】- 提供什么数据
- 【分类知识库】- 参考的分类体系
- 【分类规则】- 判断的优先级
- 【严格输出要求】- 输出格式约束

## 🎓 技术要点

### 为什么这样优化有效？

1. **知识库匹配** - AI 可以直接查找已知的网站域名
2. **规则优先级** - 明确告诉 AI 先查域名，再看标题
3. **分类体系化** - 提供完整的分类框架，减少随意性
4. **示例引导** - 每个分类都有具体网站示例

### 提示词设计原则

✅ **好的提示词：**
- 有明确的任务定义
- 有详细的分类知识库
- 有清晰的判断规则
- 有严格的输出约束

❌ **避免的做法：**
- 让 AI 自由发挥
- 不提供分类参考
- 没有判断优先级
- 输出格式不明确

## 🔄 如何使用

### 自动应用

优化后的提示词会在以下场景自动生效：

1. **添加书签时** - 自动分类建议
2. **右键菜单 AI 优化** - 手动触发优化
3. **批量导入书签** - 批量分类

### 自定义提示词

如果用户想修改提示词：

1. 打开设置页面 → AI 设置
2. 找到"智能分组"
3. 点击"编辑提示词"
4. 修改后保存

### 恢复默认

如果自定义提示词效果不好：

1. 在编辑提示词对话框
2. 点击"恢复默认"按钮
3. 确认保存
