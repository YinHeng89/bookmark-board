```json
{

  "name": "__MSG_appName__",
  "description": "__MSG_appDesc__",
  "version": "3.2.5",
  "default_locale": "zh_CN",

    "https://*/*"

  "background": {
    "service_worker": "js/background.js"
  },
  "side_panel": {
    "default_path": "sidebar.html"

    "48": "icons/logo-48.png",
```

## 📦 打包扩展

### 方法一：Chrome 开发者模式打包

1. 打开 Chrome 浏览器
2. 访问 `chrome://extensions/`
3. 开启「开发者模式」（右上角）
4. 点击「打包扩展程序」
5. 选择项目根目录（包含 manifest.json 的文件夹）
6. 私密密钥文件：**不选择**（首次打包）
7. 点击「打包」

**生成文件**：
- `bookmark-board.crx` - 扩展包文件
- `bookmark-board.pem` - 私钥文件（**重要！妥善保管**）

### 方法二：命令行打包（推荐）

```bash
# 进入项目目录
cd /Users/yinheng/Documents/GitHub/bookmark-board

# 使用 Chrome 打包工具
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome \
  --pack-extension=/Users/yinheng/Documents/GitHub/bookmark-board \
  --pack-extension-key=/path/to/your/key.pem
```

**注意**：
- 首次打包不需要 `--pack-extension-key`
- 更新版本时需要使用相同的 `.pem` 文件

#### 权限说明

**必须解释每个权限的用途**：

| 权限 | 用途说明 |
|------|---------|
| `storage` | 本地存储书签数据，确保数据持久化保存 |
| `contextMenus` | 添加右键菜单，方便快速添加书签 |
| `tabs` | 获取当前标签页信息，用于一键添加当前页面 |
| `scripting` | 注入脚本获取网页标题，提升用户体验 |
| `sidePanel` | 提供侧边栏功能，快速访问书签 |

**无需 host_permissions** - 扩展页面和 service worker 自带跨域能力

**隐私政策**（必须提供）：

创建一个简单的隐私政策页面（可以放在 GitHub）：


## ⏰ 审核流程

### 审核时间

- **首次提交**：通常 1-3 个工作日
- **更新版本**：通常几小时到 1 天
- **高峰期**：可能需要更长时间

### 审核状态

1. **待审核**（In Review）
2. **审核通过**（Published）
3. **被拒绝**（Rejected）- 需要修改后重新提交

### 常见拒绝原因

❌ **图标不符合规范**
- 解决：确保图标清晰、尺寸正确

❌ **描述包含不当内容**
- 解决：移除促销语言、外部链接

❌ **权限说明不清晰**
- 解决：详细解释每个权限的用途

❌ **隐私政策缺失**
- 解决：添加完整的隐私政策

❌ **功能与描述不符**
- 解决：确保描述准确反映功能

## 📞 问题排查

### Q: 打包失败怎么办？

A: 检查 manifest.json 格式是否正确，确保所有文件路径正确。

### Q: 审核被拒绝怎么办？

A: 查看拒绝原因，修改后重新提交。常见原因：图标不规范、描述不清晰、权限说明不足。

### Q: 如何更新已发布的扩展？

A: 
1. 更新代码
2. 修改 manifest.json 版本号
3. 使用相同的 .pem 文件重新打包
4. 在开发者控制台上传新版本

### Q: 可以修改已发布的信息吗？

A: 可以。在开发者控制台编辑商品信息，修改后需要重新审核。
